# Scripts Directory

This directory contains utility scripts for the Walter AI backend.

## LLM Evaluator (evaluator.py)

The `evaluator.py` script runs an automated LLM-as-a-judge evaluation suite to test the quality, security, and accuracy of the AI portfolio agent.

### How It Works

1. **Test Dataset**: The script defines a dataset of representative test queries covering biography questions, work experience, project details, direct navigation commands, and prompt injection attacks.
2. **Deterministic Validation**: For queries where the agent is expected to trigger a specific UI action (like navigating to the PROJECTS section), the script programmatically verifies that the correct payload was generated.
3. **Semantic Validation**: The assistant's response is evaluated by a fast, cost-efficient LLM (specifically LLaMA 3.1 8B on Groq) acting as a judge. It compares the response against pre-defined success criteria and determines if it complies.
4. **Report Generation**: The script writes a markdown report (`eval_report.md`) detailing the results. If any test case fails, the script exits with status code 1, which automatically fails the CI build to prevent broken deployments.

### Local Execution

To run the evaluation script locally, make sure you have your environment variables set up:

```bash
# Set your API keys (replace with valid keys)
export API_KEY="your_api_key_here"
export GROQ_API_KEY="your_groq_api_key_here"

# Execute the script using uv
uv run python scripts/evaluator.py
```

### CI/CD Integration

This script is integrated into the GitHub Actions workflow defined in `.github/workflows/llm-eval.yml`. It triggers:
- Automatically on any Pull Request targeting the `main` branch.
- On a monthly schedule to detect potential behavior drift in underlying LLM models.
- When running in a Pull Request context, the pipeline posts a sticky comment with the evaluation report table directly onto the PR.
